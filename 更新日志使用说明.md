# 📝 更新日志系统使用说明

## 🎯 系统概述

本博客已集成自动更新日志管理系统，可以帮助您轻松管理和维护项目的版本更新记录。

## 📁 文件结构

```
source/
├── changelog/              # 更新日志目录
│   ├── index.md           # 更新日志主页面
│   ├── v1-1-0.md         # 版本详细日志
│   └── ...               # 其他版本日志
└── architecture/          # 项目架构文档
    └── index.md          # 架构分析页面

scripts/
└── update-changelog.js    # 自动更新脚本
```

## 🚀 使用方法

### 1. 查看当前状态
```bash
hexo changelog
```
显示当前版本和使用帮助。

### 2. 创建新版本更新日志
```bash
hexo changelog 1.2.0
```
自动创建 v1.2.0 的更新日志模板。

### 3. 同步更新日志
```bash
hexo changelog sync
```
检查并同步最新的更新内容。

## 📋 手动更新流程

### 步骤1: 创建版本日志
当有新版本发布时，运行：
```bash
hexo changelog [版本号]
```

### 步骤2: 编辑详细内容
编辑生成的版本日志文件，添加具体的更新内容：
- 🆕 新增功能
- 🐛 问题修复  
- 🎨 界面优化
- ⚡ 性能优化
- 🔧 技术改进
- 📝 文档更新

### 步骤3: 更新主页面
系统会自动更新 `changelog/index.md` 主页面。

## 🔄 自动同步机制

### 触发条件
- 手动运行同步命令
- 可扩展为 Git Hook 触发
- 可集成到 CI/CD 流程

### 同步内容
- 版本号检测
- 更新日志生成
- 主页面更新
- 菜单链接维护

## 🎨 自定义配置

### 修改模板
编辑 `scripts/update-changelog.js` 中的 `generateVersionTemplate` 方法。

### 添加新分类
在模板中添加新的更新分类，如：
- 🔒 安全更新
- 🌐 国际化
- 📱 移动端优化

### 集成 Git
可以扩展脚本读取 Git 提交信息，自动生成更新内容。

## 🔗 页面访问

- **更新日志主页**: `/changelog/`

- **版本详情**: `/changelog/v[版本号]/`

## 📊 最佳实践

### 版本号规范
使用语义化版本号：
- `1.0.0` - 主版本（重大更新）
- `1.1.0` - 次版本（新功能）
- `1.1.1` - 修订版本（问题修复）

### 更新频率
- 重大功能更新：立即记录
- 小修复：可批量记录
- 定期整理：每月汇总

### 内容质量
- 使用清晰的描述
- 添加相关链接
- 包含影响范围
- 注明兼容性信息

## 🛠️ 故障排除

### 常见问题

1. **脚本无法运行**
   - 检查 Node.js 环境
   - 确认文件权限

2. **模板生成失败**
   - 检查目录权限
   - 确认路径正确

3. **菜单不显示**
   - 检查配置文件语法
   - 重启 Hexo 服务

### 技术支持
如遇问题，请检查：
- Hexo 版本兼容性
- 主题支持情况
- 文件路径正确性
